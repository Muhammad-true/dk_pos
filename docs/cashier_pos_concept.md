# Концепция экрана кассира для `dk_pos`

## Цель
Сделать современный, понятный и профессиональный экран кассира для кафе-бургерной, который:

- ускоряет оформление заказа;
- снижает количество ошибок;
- легко обучает нового кассира;
- масштабируется под будущие модули;
- хорошо работает на планшете и большом экране.

## Основная идея
Лучшее базовое решение для POS в бургерной - это главный экран в 3 рабочих зоны:

1. зона навигации по разделам;
2. зона выбора товаров;
3. зона текущего заказа и оплаты.

Такой формат дает кассиру быстрый сценарий:

`выбрать категорию -> добавить позиции -> проверить корзину -> перейти к оплате`

## Структура экрана

### 1. Верхняя панель
Содержит служебную информацию и быстрые действия:

- название точки;
- статус смены;
- номер кассы;
- язык;
- обновление меню;
- выход из профиля.

Задача верхней панели - не мешать продаже, а давать быстрый доступ к системным действиям.

### 2. Левая панель категорий
Отвечает за понятную навигацию:

- бургеры;
- донеры;
- комбо;
- напитки;
- добавки;
- соусы;
- десерты.

Требования:

- крупные элементы;
- быстрый переход между разделами;
- видимый активный раздел;
- масштабируемость под вложенные категории.

### 3. Центральная зона каталога
Основная рабочая область кассира.

Содержит:

- поиск;
- быстрые фильтры;
- популярные позиции;
- карточки товаров;
- понятную цену;
- быстрое добавление в заказ.

Требования:

- визуальная чистота;
- минимум лишнего текста;
- крупные кликабельные карточки;
- хорошая читаемость при высокой нагрузке.

### 4. Правая панель заказа
Самая критичная часть интерфейса.

Содержит:

- список выбранных позиций;
- изменение количества;
- удаление;
- заметки к заказу;
- итог;
- быстрые действия по заказу.

Нижний блок должен быть выделен визуально:

- сумма;
- количество позиций;
- тип заказа;
- основные кнопки действий.

## UX-принципы

### Скорость
- не более 2-5 касаний для типичного заказа;
- быстрые действия для популярных товаров;
- минимум сложных переходов.

### Понятность
- четкие зоны экрана;
- крупная типографика;
- простые подписи;
- визуальные акценты только на важных действиях.

### Безопасность
- защита от случайной очистки заказа;
- понятные destructive-сценарии;
- визуально различимые обычные и опасные действия.

### Масштабируемость
Экран должен быть готов к будущим функциям:

- модификаторы;
- комбо-наборы;
- скидки и промокоды;
- loyalty;
- столы;
- доставка и самовывоз;
- split payment;
- кухонный экран;
- печать чеков.

## Визуальное направление
Интерфейс должен ощущаться как профессиональный POS, а не как яркое промо-приложение.

Рекомендуемый стиль:

- темный графит или мягкий светлый нейтральный фон;
- теплый янтарный акцент;
- спокойные поверхности;
- зеленый для успешных действий;
- красный только для удаления и критичных операций.

## Этапы реализации

### Этап 1. UI mockup
Собрать современный каркас экрана на текущих данных проекта.

### Этап 2. Рабочий POS-flow
Связать UI с корзиной, категориями, состояниями заказа и оплатой.

### Этап 3. Продвинутые сценарии
Добавить модификаторы, скидки, типы заказа, заметки и быстрые шаблоны.

### Этап 4. Масштабирование
Подготовить экран к multi-branch, kitchen flow, loyalty и delivery.

## Решение для текущего этапа
На этом этапе предлагается сделать:

- профессиональную верхнюю панель;
- современную навигацию по разделам;
- улучшенные карточки товаров;
- выделенную правую панель заказа;
- блок итогов и быстрых действий;
- визуально готовую основу для следующих функций.
